# Прототипы интерфейсов

Ниже представлено текстовое описание прототипов основных интерфейсов веб-приложения, разработанное на основе функциональных требований и пользовательских историй. Описание включает ключевые элементы каждой страницы и их взаимодействие.

## 1. Страница входа/регистрации

### 1.1. Страница входа (`/login`)
*   **Заголовок:** "Вход в систему управления дефектами"
*   **Элементы:**
    *   Поле для ввода "Имя пользователя или Email"
    *   Поле для ввода "Пароль" (скрытый ввод)
    *   Кнопка "Войти"
    *   Ссылка "Забыли пароль?" (опционально, не входит в текущую область разработки)
    *   Ссылка "Зарегистрироваться" для перехода на страницу регистрации
*   **Функциональность:** Пользователь вводит учетные данные и отправляет их для аутентификации. В случае успеха, перенаправляется на Главную страницу (Dashboard). При ошибке - отображается сообщение об ошибке.

### 1.2. Страница регистрации (`/register`)
*   **Заголовок:** "Регистрация нового пользователя"
*   **Элементы:**
    *   Поле для ввода "Имя пользователя" (уникальное)
    *   Поле для ввода "Email" (уникальный)
    *   Поле для ввода "Пароль" (скрытый ввод)
    *   Поле для ввода "Повторите пароль" (скрытый ввод)
    *   Кнопка "Зарегистрироваться"
    *   Ссылка "Уже есть аккаунт? Войти" для перехода на страницу входа
*   **Функциональность:** Пользователь вводит данные, система проверяет их уникальность и создает новую учетную запись. В случае успеха, перенаправляется на страницу входа или Главную страницу. При ошибке - отображается сообщение об ошибке.

## 2. Главная страница (Dashboard) (`/dashboard`)
*   **Заголовок:** "Панель управления"
*   **Доступ:** После успешного входа в систему.
*   **Элементы:**
    *   **Верхняя навигационная панель:**
        *   Логотип системы
        *   Ссылки на основные разделы: "Проекты", "Дефекты", "Отчеты" (для Менеджеров/Наблюдателей)
        *   Выпадающее меню пользователя (имя пользователя, роль, "Мой профиль", "Выход")
    *   **Сводная информация (для всех ролей):**
        *   Краткая статистика: общее количество дефектов, дефектов в работе, закрытых дефектов.
        *   Список последних активных дефектов (например, 5-10 штук) с их статусами и проектами.
    *   **Для Менеджеров/Наблюдателей:**
        *   Графики: "Дефекты по статусам", "Дефекты по приоритетам", "Дефекты по исполнителям" (визуализация аналитических отчетов).
        *   Краткий обзор проектов: количество проектов, активных проектов.
*   **Функциональность:** Предоставляет быстрый обзор состояния системы. Интерактивные элементы (графики, списки) могут вести на соответствующие страницы с детальной информацией.

## 3. Страница списка проектов (`/projects`)
*   **Заголовок:** "Список проектов"
*   **Доступ:** Для всех аутентифицированных пользователей.
*   **Элементы:**
    *   Верхняя навигационная панель (как на Dashboard).
    *   Кнопка "Создать проект" (только для Менеджеров).
    *   Поле поиска по названию проекта.
    *   Таблица/Список проектов со следующими колонками:
        *   Название проекта (ссылка на детали проекта)
        *   Описание
        *   Дата создания
        *   Владелец
        *   Количество дефектов (общее, активные)
        *   Действия (только для Менеджеров: "Редактировать", "Удалить")
    *   Пагинация для большого количества проектов.
*   **Функциональность:** Просмотр, поиск и фильтрация проектов. Менеджеры могут создавать, редактировать и удалять проекты.

## 4. Страница деталей проекта (`/projects/[id]`)
*   **Заголовок:** "Детали проекта: [Название проекта]"
*   **Доступ:** Для всех аутентифицированных пользователей, имеющих доступ к проекту.
*   **Элементы:**
    *   Верхняя навигационная панель.
    *   Кнопки "Редактировать проект", "Удалить проект" (только для Менеджеров).
    *   **Основная информация о проекте:**
        *   Название, описание, дата создания, владелец.
    *   **Разделы (вкладки или отдельные блоки):**
        *   **Дефекты проекта:** Список дефектов, относящихся к текущему проекту (с возможностью фильтрации, сортировки, поиска и создания нового дефекта в рамках этого проекта).
        *   **Этапы проекта (опционально, в зависимости от дальнейшего развития):** Список этапов проекта.
*   **Функциональность:** Детальный просмотр информации о проекте, управление дефектами в рамках проекта.

## 5. Страница списка дефектов (`/defects`)
*   **Заголовок:** "Список дефектов"
*   **Доступ:** Для всех аутентифицированных пользователей.
*   **Элементы:**
    *   Верхняя навигационная панель.
    *   Кнопка "Создать дефект" (только для Инженеров/Менеджеров).
    *   **Панель фильтров и поиска:**
        *   Поле поиска по заголовку/описанию.
        *   Выпадающие списки для фильтрации по: Проекту, Статусу, Приоритету, Исполнителю, Репортеру.
        *   Поля выбора диапазона дат: "Дата создания от/до", "Срок выполнения от/до".
        *   Кнопка "Применить фильтры", "Сбросить фильтры".
    *   Таблица/Список дефектов со следующими колонками:
        *   ID дефекта
        *   Заголовок (ссылка на детали дефекта)
        *   Описание (краткое)
        *   Приоритет
        *   Статус
        *   Срок выполнения
        *   Исполнитель
        *   Проект
        *   Действия (только для Репортеров/Исполнителей/Менеджеров: "Редактировать", "Изменить статус", "Удалить")
    *   Пагинация.
*   **Функциональность:** Просмотр, поиск, фильтрация и сортировка дефектов. Возможность создания, редактирования и изменения статусов дефектов.

## 6. Страница деталей дефекта (`/defects/[id]`)
*   **Заголовок:** "Детали дефекта: [Заголовок дефекта]"
*   **Доступ:** Для всех аутентифицированных пользователей, имеющих доступ к дефекту.
*   **Элементы:**
    *   Верхняя навигационная панель.
    *   Кнопки "Редактировать дефект", "Изменить статус", "Удалить дефект" (видимость зависит от прав пользователя).
    *   **Основная информация о дефекте:**
        *   Заголовок, описание, приоритет, текущий статус.
        *   Информация о создании (дата, репортер), обновлении (дата) и сроке выполнения.
        *   Проект, к которому относится дефект.
        *   Назначенный исполнитель.
    *   **Разделы (вкладки или отдельные блоки):**
        *   **Комментарии:** Список комментариев с датой, автором и текстом. Форма для добавления нового комментария. Кнопки "Редактировать", "Удалить" для своих комментариев (или для менеджера).
        *   **Вложения:** Список вложенных файлов (имя файла, дата загрузки, загрузивший). Кнопка "Добавить вложение". Возможность скачать/просмотреть вложение. Кнопки "Удалить" для своих вложений (или для менеджера).
        *   **История изменений:** Список всех изменений дефекта (кто, что, когда изменил).
*   **Функциональность:** Детальный просмотр дефекта, взаимодействие с комментариями и вложениями, изменение статуса и редактирование дефекта.

## 7. Страница отчетов (`/reports`)
*   **Заголовок:** "Отчеты и аналитика"
*   **Доступ:** Только для Менеджеров и Наблюдателей.
*   **Элементы:**
    *   Верхняя навигационная панель.
    *   **Панель фильтров для отчетов:**
        *   Фильтры по: Проекту, Статусу, Приоритету, Исполнителю, Репортеру, Датам создания/срока выполнения (как на странице списка дефектов).
        *   Кнопка "Применить", "Сбросить".
    *   **Разделы отчетов:**
        *   **Аналитические графики:**
            *   Гистограмма "Количество дефектов по статусам".
            *   Круговая диаграмма "Распределение дефектов по приоритетам".
            *   Гистограмма "Количество дефектов по исполнителям".
            *   Линейный график "Динамика создания и закрытия дефектов за период".
        *   **Кнопка "Экспортировать в CSV/Excel"** (применяются текущие фильтры).
*   **Функциональность:** Просмотр визуализированных аналитических данных и экспорт детальных отчетов.
